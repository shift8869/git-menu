 git-menu.sh 的 Bash 脚本，它是一个基于命令行的 Git 操作交互式菜单。脚本的核心功能是为用户提供一个简单的界面，以执行几种常见的 Git 相关操作。

脚本的主要内容和功能如下：

1.  主菜单：程序启动后会显示一个主菜单，包含以下五个选项：
    ◦   1. 创建仓库

    ◦   2. 提交代码

    ◦   3. 回退版本

    ◦   4. 拉取仓库

    ◦   0. 退出

2.  功能详解：
    ◦   创建仓库 (选项1)：

        ▪   引导用户输入一个 GitHub 仓库名（如 shift8869/WHMES.git）。

        ▪   在当前目录初始化一个新的 Git 仓库（如果还不是仓库的话）。

        ▪   自动添加并提交工作区中已有的更改，初始提交信息为“first commit”。

        ▪   将默认分支重命名为 main。

        ▪   配置远程仓库地址 (origin)。如果已存在，会询问用户是否更新。

        ▪   检查远程仓库是否存在，如果不存在，会询问用户是否通过 gh (GitHub CLI) 自动创建一个私有远程仓库。

        ▪   最后，将本地仓库推送到远程并建立追踪关系。

    ◦   提交代码 (选项2)：

        ▪   引导用户输入本次提交的更新说明。

        ▪   自动执行 git add .，然后使用用户提供的提交信息进行 git commit。

        ▪   最后执行 git push 将提交推送到远程仓库。

    ◦   回退版本 (选项3)：

        ▪   显示最近的 10 条提交记录。

        ▪   让用户从列表中选择一个序号，以指定要回退到的版本。

        ▪   在用户确认后，执行 git reset --hard 将本地仓库强制回退到指定版本，并使用 git push -f 强制更新远程仓库。

    ◦   拉取仓库 (选项4)：

        ▪   引导用户输入一个 GitHub 仓库名。

        ▪   自动将 https://github.com/[repo_name] 的仓库克隆到本地，目录名默认为仓库名（去除 .git 后缀）。

3.  脚本特点：
    ◦   包含基本的输入验证和错误处理。

    ◦   在执行具有破坏性或覆盖性操作（如设置远程仓库、强制回退）前，通常会请求用户确认。

    ◦   程序启动时会先显示当前工作目录，便于用户确认位置。

    ◦   它是一个交互式的命令行工具，旨在通过简单的菜单引导用户完成一系列标准的 Git 工作流操作。
